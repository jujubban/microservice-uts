Pengerjaan UTS Microservices ini diawali dengan instalasi Laragon dan ActiveMQ Classic sebagai message broker yang berjalan di port 61614 untuk WebSocket, lalu membuat dua service menggunakan Node.js dimana Service 1 (Producer) berfungsi menerima pendaftaran user melalui form dan mengirimkan data ke queue registration_queue tanpa mengirim email langsung, sementara Service 2 (Consumer) berjalan sebagai worker yang memantau queue dan mensimulasikan pengiriman email ke terminal. Setelah itu dilakukan pengujian asinkron dengan mematikan consumer lalu mendaftarkan 5 user yang mengakibatkan pesan tertahan di ActiveMQ (terlihat di dashboard), kemudian consumer dinyalakan kembali dan kelima pesan terproses otomatis membuktikan sifat resilience message queue. Tampilan front-end ditingkatkan dengan desain modern menggunakan gradient background, animasi, ikon, dan live validation. Seluruh kode, screenshot dokumentasi, serta video demo berdurasi 3 menit diupload ke GitHub repository microservice-uts dengan README yang lengkap sebagai bukti penyelesaian tugas.
